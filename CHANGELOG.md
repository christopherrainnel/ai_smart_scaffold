# Changelog

All notable changes to AI Smart Scaffold public releases are documented in this file.

The format follows Keep a Changelog and Semantic Versioning.

## [0.1.0] - 2026-04-01

### Added

- First public release line for AI Smart Scaffold
- VS Code extension ships first in the public release rollout, with Workbench and broader runtime surfaces following after the VS Code release
- Early users who begin on or before June 30, 2026 start in Tier1 mode for up to 20 days, with all bundled introductory access ending no later than June 30, 2026, before returning to the Free baseline unless a separate paid Tier1 entitlement is active
- Shared notification history across connected product surfaces so guided actions and runtime alerts stay visible instead of disappearing with one dismissed toast
- Repo-scoped change alerts for important created, moved, or deleted files detected outside the active governed session
- Public installation, support, legal, and release documentation

### Changed

- Public docs and pricing copy now explain the runtime in human terms: it acts like an orchestrator between you, your AI tools, and your repo to reduce drift and keep guided work visible
- Customer-facing copy now explains Tier1 as a more guided experience for completing real projects, not just a larger rule set
- Product messaging now makes it clearer that project-authored markdowns stay editable in the same workspace

### Notes

- AI Smart Scaffold remains local-first. Repository-specific project work stays on the installed machine unless the user deliberately submits support material or connects to an optional hosted service.
