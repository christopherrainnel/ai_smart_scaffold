# AI Smart Scaffold 0.1.0

> Launch note: The VS Code extension is the first shipped surface in this public release. The Workbench desktop app and broader runtime surfaces follow after the VS Code rollout.

Version `0.1.0` is the first public release line for AI Smart Scaffold.

## Release Focus

This release establishes the initial public product surface for:

- the VS Code extension-first rollout
- installation guidance
- system requirements
- supported-tool guidance
- legal and security documents
- release notes and asset references
- the first-install Tier1 introductory workflow

## Customer-Facing Highlights

- AI Smart Scaffold now explains its role in plain language: it acts like the orchestrator between you, your AI tools, and your repo to reduce drift.
- As connected product surfaces are used together, product-only notifications stay visible instead of disappearing after a single toast.
- Important created, moved, or deleted files outside the active session surface as repo notices instead of being pinned to the wrong chat.
- Your own project markdowns stay editable in the same workspace while the runtime surfaces follow-up when managed areas need review.
- Tier1 is positioned as deeper guided help for finishing real projects when AI output starts to sprawl.

## Included Product Forms

- VS Code extension
- Workbench desktop app
- packaged governed runtime components

## Notes

AI Smart Scaffold is designed as a local-first product. Repository-specific work remains local unless support material is deliberately shared by the user.

For `0.1.0`, the product keeps the same workspace on the Free baseline until the initial entitlement sync completes, then moves eligible first installations into Tier1 in that same workspace for up to 20 days, with all bundled introductory access ending no later than June 30, 2026. After that introductory window ends, the workspace returns to Free unless a separate paid Tier1 entitlement is active.

This release also keeps machine-safe runtime protection active across tiers: the current runtime starts from a 3-session active baseline, can temporarily lift the soft ceiling to 4 when the machine has healthy headroom, and uses context budgets to prevent prompt flooding and overloaded long sessions.
