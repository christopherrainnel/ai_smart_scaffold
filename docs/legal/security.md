# Security Policy

## Reporting A Vulnerability

If you discover a vulnerability in AI Smart Governance, please do not open a public issue.

Email: `inquiry@hsnexus.com`

Useful report details include:

- A short description of the vulnerability and potential impact
- A minimal reproduction path
- Product surface affected, such as VS Code extension, plugin, workbench, CLI/MCP, account service, billing, or entitlement
- Version or release channel, if known
- Any suggested mitigation

Avoid sending secrets, service-role keys, private repositories, customer data, payment card details, or large diagnostic bundles unless H&S Nexus Ventures gives you a safer transfer method.

## Public Scope

This public repository is in scope for documentation and issue-intake security issues. The private runtime, extension, workbench, backend, account, and entitlement services are operated from internal repositories and official deployment surfaces, but reports about those products may still be sent to the same contact address.

## Principles

- Account-required official governance
- Server-authoritative entitlement and quota decisions
- Least-privilege backend credentials
- No public exposure of service-role keys or private signing keys
- No public issue disclosure for bypass techniques before coordinated review
